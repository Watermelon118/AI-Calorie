# AI-Calorie Agent Instructions

本文件用于给新的 Codex/AI 代理会话提供仓库级上下文。开始处理本项目时，先读取本文件，再读取 `PROGRESS.md`。

## 项目上下文

- 项目名称：AI-Calorie。
- 应用语言：中文。
- 应用目标：记录一日三餐、每顿热量、每日统计和热量目标。
- 当前开发顺序：先做前端，再固定请求路径和接口契约，最后做后端。
- 前端技术：React + Ant Design Mobile。
- 后端技术：C# + ASP.NET Web API，当前暂缓开发。

## 必读文件

- `PROGRESS.md` 是唯一的项目进度跟踪入口。
- 开始任何新任务前，先用 UTF-8 编码读取 `PROGRESS.md`，确认当前阶段、已完成内容和下一步。
- 完成实际开发动作后，同步更新 `PROGRESS.md`，只修改已有章节或补充必要决策，不重复新增同类内容。

## 编码要求

- 读取中文文件时显式使用 UTF-8。
- 写入 Markdown、前端文案、接口说明等中文内容时保持 UTF-8。

## 开发约定

- 前端页面和交互逻辑优先，不提前实现 ASP.NET 后端业务。
- 前端可以先使用本地 mock 数据或轻量状态管理完成页面闭环。
- 请求路径和接口契约在前端页面逻辑稳定后统一整理。
- 后端阶段应以 `PROGRESS.md` 中最终确认的 API 契约为准。

## 工作方式

- 保持改动聚焦，不做无关重构。
- 有新增页面、路由、数据模型、接口草案或关键决策时，更新 `PROGRESS.md`。
- 如果新会话缺少上下文，优先从 `AGENTS.md` 和 `PROGRESS.md` 恢复项目状态，不要求用户重新描述进度。

## Git 约定

- Git 相关命名和说明统一使用英文，包括 commit message、branch name、tag、PR 标题和 PR 描述。
